# development-process
開発工程とは

## 要件定義
* 何がしたいのか → 目標設定
* そのためにどうすれば良いか
  * ユーザーや関係者とのヒアリング
  * 機能要件・非機能要件の整理

## 設計
* どのように実現するか　→ 目標達成手法検討
  * 画面設計（UI/UX）
    * どのような操作で、どのような結果を得たいか
  * システム構成設計、データ設計、API設計など
    * どのような環境で動かすことになるか
    * どの開発環境・言語を使うのが適切か

## 実装
* 実体化する
  * プログラミング
  * 単体テスト
    * 不具合修正

## 評価
* やりたいことができたか、何をもってできたことを証明するか　→ 目標達成確認
  * 統合テスト
    * 不具合修正
    * パフォーマンス確認

## 保守・運用
* やりたいことを実践し、不満点や改良点を見つける
  * 本番環境への公開
  * 保守・アップデート
    * 引き継ぎ
    * 資料作成
